- `sds.c` — has a `#ifdef HI_SDS_TEST_MAIN`-guarded `main()` that gazelle detects and misclassifies as a binary source; kept in library via `# keep`
- `dict.c` — included directly via `#include "dict.c"` from async.c; must be `textual_hdrs`, not a separate compile unit; excluded from gazelle's scan so it doesn't appear in `srcs`

### Biggest blocker (encountered and solved)
Gazelle's `findEmptyRules` deletes any `cc_test` or `cc_library`-family rule whose source files aren't found in the scanned file list. The solution was two-fold: (1) keep `hiredis_test.c` visible to gazelle (so it appears in fileInfos and the cc_test is not flagged for deletion), and (2) use `# keep` on the `sds.c` srcs entry so gazelle preserves it even though sds.c itself is excluded from gazelle's classification (to prevent the broken cc_binary from being generated).
